‚úÖ Semantic Class Names

Names describe purpose (bulb-glow) not appearance (cream-color).

CSS Custom Properties - 7 Variables Control 200+ Paths!

Remember CSS custom properties from the Colors lab? Prof. Teeters taught me I could use them in SVG too! I built my entire Lightbulb around just 7 variables. This was the moment everything clicked for me! ü§Ø

HAP's Lightbulb Color System

This is all it takes to control 200+ paths:

:root {
  /* HAP's Lightbulb Color System */
  --bulb-glow: hsl(45, 100%, 89%);      /* Lightest - the emanating light */
  --bulb-warm: hsl(40, 97%, 73%);       /* Medium - warm glass tones */
  --bulb-bright: hsl(34, 100%, 65%);    /* Intense - the hot filament */
  --metal-light: hsl(60, 4%, 74%);      /* Highlight on metal */
  --metal-medium: hsl(60, 2%, 57%);     /* Body of metal base */
  --metal-dark: hsl(0, 1%, 49%);        /* Shadow on metal */
  --base-dark: hsl(210, 4%, 21%);       /* Darkest - grounding element */
}

These 7 variables connect to my 7 semantic groups. Each variable controls dozens of paths! üü†

Connecting Variables to Classes

Here's how I transformed my hardcoded Station 3 CSS to use variables:

üü† HAP's Old Way vs. What I Learned

‚ùå WRONG: Station 3 Hardcoded Approach

.bulb-glow { fill: #ffefcb; }
.bulb-warm { fill: #fdcf74; }
.bulb-bright { fill: #ffb24f; }
/* Locked in! Can't theme! */

‚úÖ RIGHT: Station 4 Variable Approach

.bulb-glow { fill: var(--bulb-glow); }
.bulb-warm { fill: var(--bulb-warm); }
.bulb-bright { fill: var(--bulb-bright); }
/* Themeable! Just change the variables! */

The classes stay the same. The HTML stays the same. Only the variable definitions change. That's the power of systematic organization! üü†

Quick Reference

1

Organization First

CSS variables only work well with good organization. Station 3's semantic groups made Station 4's theming possible!

2

Define in :root

Put your color variables in :root { } so they're available throughout your SVG.

3

Use var() in Classes

Connect variables to classes: .bulb-glow { fill: var(--bulb-glow); }

4

Semantic Variable Names

Name variables by purpose (--bulb-glow) not appearance (--cream-color).

5

Theme with Class Names

Switch themes by adding classes: .lightbulb.theme-cool overrides the default variables.

6

Test Your System

Change one variable and watch dozens of paths update. If they don't, check your organization!
