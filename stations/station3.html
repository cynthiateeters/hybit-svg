Three Ways to Style SVG (I Tried Them All!)

Prof. Teeters showed me there are three different methods for styling SVG elements. I experimented with all three on my Lightbulb before finding the best approach for my needs.

Method 1: Inline Attributes

My First Mistake: This is how I initially created my Lightbulb:

<path fill="#ffefcb" d="M168.5 660.1..."/>
<path fill="#ffefcb" d="M789.5 660.1..."/>
<!-- I typed this 38 more times! -->

When to use: Quick one-off graphics, prototyping, when you need only one or two elements styled

Cons: Repetitive (my 40+ glow paths!), hard to maintain, no reusability

Method 2: Internal <style> Tag

The Lightbulb's Evolution: This is how I reorganized my Lightbulb:

<svg>
  <style>
    /* Regular CSS classes with direct color values */
    .bulb-glow { fill: #ffefcb; }
    .bulb-warm { fill: #fdcf74; }
    .metal-light { fill: #bdbdbb; }
  </style>

  <g class="bulb-glow">
    <path d="M168.5 660.1..."/>
    <!-- All 40+ glow paths! -->
  </g>
</svg>

HAP's choice: I used internal styles for my Lightbulb so it's self-contained but still organized. Anyone can use it without needing external files!

Notice I'm using regular CSS with hardcoded colors. In Station 4, I'll show you how CSS custom properties make these colors dynamic and themeable! üü†

Method 3: External CSS

For Design Systems: Prof. Teeters showed me this approach for when multiple SVGs share styles:

/* In your stylesheet */
.lightbulb .bulb-glow {
  fill: #ffefcb;
}
.lightbulb .metal-parts {
  fill: #bdbdbb;
}

When to use: Multiple SVGs sharing styles, building a design system, maintaining consistency across graphics

Groups (<g>) - My Organization Superpower

I had 200+ paths floating around. It was chaos! I couldn't figure out which paths created the glow versus the metal parts. Prof. Teeters showed me how to use the <g> (group) element to organize them by visual purpose.

How I Organized My Lightbulb:

Instead of 200+ unorganized paths, I grouped them by their visual purpose:

<!-- HAP's Lightbulb Organization -->
<g class="bulb-glow">
  <!-- All 40+ paths that create the glow effect -->
</g>
<g class="bulb-warm">
  <!-- All paths for the warm middle tones -->
</g>
<g class="bulb-bright">
  <!-- The hot filament paths -->
</g>
<g class="metal-light">
  <!-- All paths for metal highlights -->
</g>
<g class="metal-medium">
  <!-- Metal body paths -->
</g>
<g class="metal-dark">
  <!-- Metal shadow paths -->
</g>
<g class="base-dark">
  <!-- The darkest base elements -->
</g>

Now instead of 200+ unorganized paths, I have 7 logical groups! Each group has a purpose, a name, and can be styled as a unit. üü†

Semantic Class Names I Chose

I named my classes by what they DO, not what color they ARE. Prof. Teeters explained why this matters:

.bulb-glow

NOT .light-yellow

The light emission effect

.bulb-warm

NOT .medium-yellow

The warm glass tones

.bulb-bright

NOT .orange

The hot filament

.metal-light

NOT .light-gray

Metal highlights

.metal-medium

NOT .gray

Metal body

.metal-dark

NOT .dark-gray

Metal shadows

.base-dark

NOT .very-dark-gray

The grounding element

üéì HyBit's Quick Reference

1

Start with Groups, Not Colors

Organize your SVG into logical <g> groups first. I grouped my 200+ paths into 7 semantic groups based on visual purpose. Structure first, styling second!

2

Name by Purpose, Not Appearance

Use .bulb-glow not .yellow. Use .metal-shadow not .dark-gray. Semantic names work with ANY color scheme and make your code self-documenting.

3

Move Styles Out of Inline Attributes

Remove all fill="" and stroke="" attributes from your SVG elements. Move them to a <style> block using CSS classes. This makes your SVG code cleaner and easier to understand!

4

Test Your Organization

Can you easily find all the parts that serve a similar purpose? If your metal parts are scattered throughout the code, reorganize them into one group. Good organization means related elements are together!
